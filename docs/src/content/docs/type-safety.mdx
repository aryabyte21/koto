koto can generate TypeScript type definitions from your translation files, giving you autocompletion and compile-time safety for every translation key in your app.

## Why type-safe translation keys?

Without types, translation key errors are silent runtime bugs:

```ts
// Typo — renders empty string or the raw key in production
t('homePage.hero.subtitel')
//                  ^^^^^ no error, no warning
```

With koto-generated types, your editor and CI catch this immediately:

```ts
// TypeScript error: Property 'subtitel' does not exist
t('homePage.hero.subtitel')
//                ^^^^^ TS2339
```

## Generating types

<Steps>

1. **Enable type generation** in your config:

   ```ts
   // koto.config.ts
   import { defineConfig } from 'koto'

   export default defineConfig({
     sourceLocale: 'en',
     targetLocales: ['es', 'fr', 'de'],
     files: ['src/locales/[locale].json'],
     provider: {
       name: 'openai',
       model: 'gpt-4o-mini',
     },
     types: {
       output: 'src/types/i18n.d.ts',
       framework: 'next-intl',
     },
   })
   ```

2. **Run the types command:**

   ```bash
   npx koto types
   ```

   ```
   ✔  Generated type definitions → src/types/i18n.d.ts
      87 keys mapped
   ```

3. **Include the generated file** in your `tsconfig.json` if needed (most setups pick it up automatically).

</Steps>

## Generated type file

Given a source locale file like:

```json
{
  "common": {
    "save": "Save",
    "cancel": "Cancel",
    "delete": "Delete"
  },
  "homePage": {
    "hero": {
      "title": "Welcome to {appName}",
      "subtitle": "The best way to manage your projects"
    }
  },
  "auth": {
    "login": "Log in",
    "signup": "Sign up",
    "forgotPassword": "Forgot password?"
  }
}
```

koto generates a type definition file that maps every key path:

```ts
// src/types/i18n.d.ts
// Auto-generated by koto — do not edit manually

export interface Messages {
  common: {
    save: string
    cancel: string
    delete: string
  }
  homePage: {
    hero: {
      title: string
      subtitle: string
    }
  }
  auth: {
    login: string
    signup: string
    forgotPassword: string
  }
}
```

## Framework integrations

<Tabs>
  <TabItem label="next-intl">
    Set `framework: 'next-intl'` in your types config. koto generates a module declaration compatible with next-intl's `useTranslations` hook:

    ```ts
    // koto.config.ts
    types: {
      output: 'src/types/i18n.d.ts',
      framework: 'next-intl',
    }
    ```

    Generated output:

    ```ts
    // src/types/i18n.d.ts
    import 'next-intl'

    type Messages = {
      common: { save: string; cancel: string; delete: string }
      homePage: { hero: { title: string; subtitle: string } }
      auth: { login: string; signup: string; forgotPassword: string }
    }

    declare module 'next-intl' {
      interface AppMessages extends Messages {}
    }
    ```

    Now `useTranslations('common')` gives you autocompletion for `save`, `cancel`, and `delete`.
  </TabItem>
  <TabItem label="i18next">
    Set `framework: 'i18next'`:

    ```ts
    types: {
      output: 'src/@types/i18next.d.ts',
      framework: 'i18next',
    }
    ```

    Generated output:

    ```ts
    // src/@types/i18next.d.ts
    import 'i18next'

    declare module 'i18next' {
      interface CustomTypeOptions {
        defaultNS: 'translation'
        resources: {
          translation: {
            'common.save': string
            'common.cancel': string
            'common.delete': string
            'homePage.hero.title': string
            'homePage.hero.subtitle': string
            'auth.login': string
            'auth.signup': string
            'auth.forgotPassword': string
          }
        }
      }
    }
    ```
  </TabItem>
  <TabItem label="react-intl">
    Set `framework: 'react-intl'`:

    ```ts
    types: {
      output: 'src/types/intl.d.ts',
      framework: 'react-intl',
    }
    ```

    Generated output:

    ```ts
    // src/types/intl.d.ts
    import 'react-intl'

    declare module 'react-intl' {
      interface AppMessageIds {
        'common.save': string
        'common.cancel': string
        'common.delete': string
        'homePage.hero.title': string
        'homePage.hero.subtitle': string
        'auth.login': string
        'auth.signup': string
        'auth.forgotPassword': string
      }
    }
    ```
  </TabItem>
  <TabItem label="Generic">
    Set `framework: 'generic'` (or omit `framework`) for a plain interface export:

    ```ts
    types: {
      output: 'src/types/i18n.d.ts',
      framework: 'generic',
    }
    ```

    This generates a simple `Messages` interface you can use however you like.
  </TabItem>
</Tabs>

## Watch mode

During development, run koto types in watch mode to regenerate types whenever your source locale files change:

```bash
npx koto types --watch
```

```
◆  Watching for changes...
   src/locales/en.json changed → regenerating types
✔  Generated type definitions → src/types/i18n.d.ts
   92 keys mapped
```

This pairs well with your dev server for a seamless workflow: edit source strings, and your types update automatically.

## Tips

- **Commit the generated type file** to your repo so all team members and CI have access to it without running `koto types` first.
- **Add `koto types` to your build script** as a pre-build step to ensure types are always in sync:
  ```json
  {
    "scripts": {
      "prebuild": "koto types",
      "build": "next build"
    }
  }
  ```
- **Use `koto types --watch`** alongside your dev server for instant feedback on new or renamed keys.
