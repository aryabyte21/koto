## Why koto?

Most i18n tools treat translation as a flat key-value lookup. koto understands that "Submit" on a payment form and "Submit" on a feedback widget need different translations. Context profiles let you define tone, terminology, and instructions per section of your app — so every string lands right.

<CardGrid>
  <Card title="Context Profiles" icon="pencil">
    Define different tones for different parts of your app. Marketing pages get punchy copy. Legal pages stay formal. UI strings stay concise.
  </Card>
  <Card title="BYOLLM" icon="setting">
    Bring your own LLM. Works with OpenAI, Anthropic Claude, Google Gemini, and local models via Ollama. Switch providers with one config line.
  </Card>
  <Card title="Type Safety" icon="approve-check">

## Get started in 30 seconds

```bash
# Install
pnpm add -D koto

# Interactive setup — detects your framework and locale files
npx koto init

# Translate everything
npx koto translate
```

## Minimal configuration

```ts
// koto.config.ts
import { defineConfig } from 'koto'

export default defineConfig({
  sourceLocale: 'en',
  targetLocales: ['es', 'fr', 'de', 'ja'],
  files: ['src/locales/[locale].json'],
  provider: {
    name: 'openai',
    model: 'gpt-4o-mini',
  },
})
```

That's it. Run `npx koto translate` and your locale files are ready.
