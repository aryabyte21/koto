## Why koto?

<CardGrid>
  <Card title="Context Profiles" icon="pencil">
    Assign tone and instructions by file pattern. Marketing copy sounds energetic. Legal text stays formal. One command handles all of it.
  </Card>
  <Card title="Bring Your Own LLM" icon="setting">
    OpenAI, Anthropic, Google Gemini, Ollama, or any OpenAI-compatible endpoint. Switch providers with one config line.
  </Card>
  <Card title="Type-Safe Keys" icon="approve-check">
    Generate TypeScript types from your locale files. Missing or misspelled keys fail at build time, not in production.
  </Card>
  <Card title="Smart Cache" icon="rocket">
    SHA-256 content hashing + lockfile. Unchanged strings are never re-translated. Follow-up runs take seconds.
  </Card>
  <Card title="Quality Scoring" icon="star">
    Placeholder integrity, length ratio, untranslated detection. Gate your CI pipeline on translation quality.
  </Card>
  <Card title="CI/CD Ready" icon="laptop">
    GitHub Action included. Use `--fail-on-error` to block releases when translations are incomplete.
  </Card>
</CardGrid>

## Minimal config

```ts
// koto.config.ts
import { defineConfig } from 'koto'

export default defineConfig({
  sourceLocale: 'en',
  targetLocales: ['es', 'fr', 'de', 'ja'],
  files: ['src/locales/[locale].json'],
  provider: { name: 'openai', model: 'gpt-4o-mini' },
})
```

That's it. Run `npx koto` and your locale files are ready.
