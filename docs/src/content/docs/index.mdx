## Build Fast Without Translation Debt

Most tools translate keys in isolation. koto translates with product context, type-aware outputs, and CI-friendly workflows so your team can treat localization like normal engineering work.

<CardGrid>
  <Card title="Context Profiles" icon="pencil">
    Assign tone and domain instructions by file pattern. Marketing can sound energetic while legal copy stays formal and precise.
  </Card>
  <Card title="BYOLLM" icon="setting">
    Bring your own model provider. OpenAI, Anthropic, Google, Ollama, or OpenAI-compatible endpoints all work behind one config interface.
  </Card>
  <Card title="Type Safety" icon="approve-check">
    Generate framework-specific TypeScript types from locale keys so missing or misspelled keys fail before production.
  </Card>
  <Card title="Smart Cache" icon="rocket">
    Incremental lockfile + hash caching means unchanged strings are skipped. Most follow-up runs complete in seconds.
  </Card>
  <Card title="Quality Scoring" icon="star">
    Every translation is scored for placeholder integrity, length ratio, and glossary consistency so quality gates can run in CI.
  </Card>
  <Card title="CI/CD Ready" icon="laptop">
    Run in GitHub Actions or any CI. Use `--fail-on-error` to block releases if translations are incomplete or below threshold.
  </Card>
</CardGrid>

## Get started in 30 seconds

```bash
# Install the CLI
pnpm add -D koto

# Interactive setup: detects framework + locale files
npx koto init

# Translate pending keys
npx koto translate
```

## Minimal configuration

```ts
// koto.config.ts
import { defineConfig } from 'koto'

export default defineConfig({
  sourceLocale: 'en',
  targetLocales: ['es', 'fr', 'de', 'ja'],
  files: ['src/locales/[locale].json'],
  provider: {
    name: 'openai',
    model: 'gpt-4o-mini',
  },
})
```

That's it. Run `npx koto translate` and your locale files are ready.
