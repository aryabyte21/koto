## Prerequisites

- **Node.js 18+** — koto uses modern Node APIs.
- **An LLM API key** — OpenAI, Anthropic, Google, or a local Ollama instance. See the [Providers guide](../providers/) for setup details.
- **Existing i18n source file** — at least one locale file (JSON, TypeScript, YAML, or PO) with your source strings.

## What just happened?

When you run `koto translate`, three things happen:

1. **Diffing** — koto reads your source locale file and compares it against each target locale. Only new or changed keys are queued for translation.

2. **Translation** — Each pending key is sent to your configured LLM provider with the appropriate context profile (tone, glossary, instructions). Translations are batched for efficiency.

3. **Output** — Translated strings are written to your target locale files. A lockfile (`koto-lock.json`) is updated with content hashes so the next run skips unchanged keys.

### Files created

| File | Purpose |
|---|---|
| `koto.config.ts` | Your project configuration |
| `koto-lock.json` | Content hashes for incremental translation (commit this) |
| `src/locales/es.json` | Translated locale files (one per target locale) |

:::tip
Commit `koto-lock.json` to your repo. It allows koto to skip unchanged keys on subsequent runs, making incremental translations fast.
:::
